    DAOS Reset¶

    Motivation: (Not sure if this should be included, but I’m writing this for my personal use as well) Often times I start an instance of DAOS, kill a PID, run a test and have a buggy DAOS instance suddenly. The following steps always place DAOS on a clean slate by resetting the metadata.

    On Server:

    $ ls /mnt/daos0/  #Find Mounting point  
a8a9f987-b782-4f33-8373-8108f074a7e9  daos_nvme.conf  NEWBORNS    ZOMBIES 
control_raft                          daos_sys        superblock 
$ sudo systemctl stop daos_server.service   #stop running DAOS instance 
$ sudo systemctl disable daos_server.service

$ umount /mnt/daos0  #unmount the metadata 
$ sudo systemctl daemon-reload   #start DAOS 
$ sudo systemctl start daos_server 
$ sudo systemctl status daos_server -l

    FIO Install and Setup with DFS¶

    Motivation: FIO is useful for testing storage performance without needing to write a lengthy test application from scratch. Allows the user to test their current workstation with ease, and the use of a simple config file given below. FIO best suited for single client runs, while other tools such as IOR and MDTEST are more suited for cluster runs. FIO is capable to leverage various I/O engines, including standard Linux access such as libaio and pvsync, while also DAOS specific dfs. Latter requires compiling FIO from a source to enable it.

    Installing FIO¶

    There are several ways to execute FIO benchmarks by running with various I/O engines configured for DAOS POSIX container use. Those are:

    1. Standard Linix engines, such as libaio and pvsync, used over dfuse implementation
    2. Standard Linix engines, such as libaio and pvsync, executed with DAOS I/O interception library
    3. DAOS I/O engine for FIO (dfs) directly submitting I/O

    If only options #1 and #2 are considered, then FIO version from Linux distro can be used. For #3 option the latest FIO required to be compiled from source (covered in next section).

    To install FIO from distro repository, simply run: 

    $ sudo yum install -y fio

    FIO requires a Config file or command line parameters to operate. The following link has many examples for different FIO tests: https://github.com/axboe/fio/tree/master/examples. The FIO Documentation is extensive and covers many different test cases to properly benchmark your respective engine here: https://fio.readthedocs.io/en/latest/fio_doc.html
    A sample FIO Config file named seqwrite_test.fio is below with a runtime limit 10 minutes:

    seqwrite_test.fio: 

    [global] 
ioengine=libaio 
directory=/tmp/daos_test1  
filename_format=fio-test.$jobnum

cpus_allowed_policy=split 
group_reporting=1 
time_based=1 
percentile_list=99.0:99.9:99.99:99.999:99.9999:100 
disable_slat=1 
disable_clat=1 
size=1G 
randrepeat=0 
norandommap 
refill_buffers 
runtime=600

[daos-seqwrite] 
name=1M Sequential Write in 1 thread 
rw=write 
bs=1M 
iodepth=16 
numjobs=1

!!! note:
    ‘direct=1’ flag not used since I/O is cached with dfuse deamon by default.  
    A potential solution is to disable the dfuse cahing first by “--disable-caching” parameter in dfuse deamon before mounting a container. See https://daos-stack.github.io/devbranch/user/filesystem/#caching

    Installing FIO with DFS support¶

    For Centos 7.9 FIO needs GCC greater than 4.8, centos 7.9 is only at 4.8. Therefore we need to use a devtoolset, I used devtoolset-7 to get the make install in the next step working.

    $ sudo yum install centos-release-scl 
$ sudo yum-config-manager --enable rhel-server-rhscl-7-rpms 
$ sudo yum install devtoolset-7 
$ scl enable devtoolset-7 bash

    In your home directory, begin the installation process for FIO below. 

    $ git clone http://git.kernel.dk/fio.git 
$ cd fio  
$ CFLAGS="-I/path/to/daos/install/include" LDFLAGS="-L/path/to/daos/install/lib64" ./configure 
$ make install

    Verify that the DFS (DAOS File System) engine is also installed, by running the command below. DFS should be visible. If not, verify that all dependencies have been installed and linked properly here (https://wiki.hpdd.intel.com/display/DAOS/DAOS+Build+from+scratch).
