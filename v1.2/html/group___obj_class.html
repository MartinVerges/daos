Predefined object classes It describes schema of data distribution & protection.

Enumerator
OC_BACK_COMPAT 

Object classes with no data protection NB: The first 50 IDs are reserved for backward compatibility

OC_TINY 

Single shard object

OC_SMALL 

Object with small number of shards. Number of shards of the class is chosen by DAOS based on the current size of the pool.

OC_LARGE 

Object with large number of shards. Number of shards of the class is chosen by DAOS based on the current size of the pool.

OC_MAX 

Object with maximum number of shards. Number of shards of the class is chosen by DAOS based on the current size of the pool.

OC_RP_TINY 

object classes protected by replication Tiny object protected by replication This object class has one redundancy group

OC_RP_SMALL 

Replicated object with small number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_LARGE 

Replicated object with large number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_MAX 

Replicated object with maximum number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_SF_TINY 

Object classes protected by replication which supports Scalable Fetch (SF) SF classes have more replicas, so they are slower on update, but more scalable on fetch because they have more replicas to serve fetches. Tiny object protected by replication This object class has one redundancy group

OC_RP_SF_SMALL 

(SF) Replicated object with small number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_SF_LARGE 

(SF) Replicated object with large number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_SF_MAX 

(SF) Replicated object with maximum number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_RP_XSF 

Replicated object class which is extremely scalable for fetch. It has many replicas so it is very slow for update.

OC_EC_TINY 

Object classes protected by erasure code Tiny object protected by EC This object class has one redundancy group

OC_EC_SMALL 

EC object with small number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_EC_LARGE 

EC object with large number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_EC_MAX 

EC object with maximum number of redundancy groups. Number of redundancy groups of the class is chosen by DAOS based on the current size of the pool.

OC_S1 

Object classes with explicit layout Object classes with explicit layout but no data protection Examples: S1 : shards=1, S2 means shards=2, ... SX : spreading across all targets within the pool

OC_RP_2G1 

Replicated object with explicit layout The first number is number of replicas, the number after G stands for number of redundancy Groups.

Examples: 2G1 : 2 replicas group=1 3G2 : 3 replicas groups=2, ... 8GX : 8 replicas, it spreads across all targets within the pool 2-way replicated object classes

OC_RP_3G1 

3-way replicated object classes

OC_RP_4G1 

4-way replicated object classes

OC_RP_6G1 

6-way replicated object classes

OC_RP_8G1 

8-way replicated object classes

OC_EC_2P1G1 

Erasure coded object with explicit layout

  • the first number is data cells number within a redundancy group
  • the number after P is parity cells number within a redundancy group
  • the number after G is number of redundancy Groups.

Examples:

  • 2P1G1: 2+1 EC object with one redundancy group
  • 4P2G8: 4+2 EC object with 8 redundancy groups
  • 8P2G2: 8+2 EC object with 2 redundancy groups
  • 16P2GX: 16+2 EC object spreads across all targets within the pool EC 2+1 object classes
